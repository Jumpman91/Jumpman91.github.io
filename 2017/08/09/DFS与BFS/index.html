广度优先搜索 Breadth First Search

基本思想

从一个顶点v开始，层层递进，对于每一个节点都访问它的所有邻接节点，并且记录该节点，访问完它的所有邻接节点后结束该节点的访问。广度优先用到了“先进先出”队列，通过这个队列来存储第一次发现的节点，以便下一次的处理；而对于再次发现的节点，不予理会

总结

  • DFS在空间上有优势，因为DFS不需要保存搜索过程中的状态，而BFS一般需要队列来记录。
  • BFS一般更适用于搜索最短路径，因为BFS搜索中遇到目标就一定是最优解，搜索就可以终止。但DFS必须全部搜索完毕，才能从所有解中找到最优解。
  • 若要得出所有解，DFS更有优势，相比于BFS易于编写（递归），且不用记录过多信息。
